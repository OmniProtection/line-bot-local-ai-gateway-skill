# LINE Bot Local AI Gateway Skill v0.1.0-alpha Release Notes Draft

Version: `v0.1.0-alpha`

Date: TBD

Status: Alpha

## Summary

Initial public alpha for `LINE Bot Local AI Gateway Skill`, a non-official Codex Skill and starter template for local-first LINE Bot AI gateways with LM Studio, SQLite memory, explicit web search, and safety gates.

`local-free-line-bot-creator` is the repository slug and legacy identifier.

## Included

- Codex Skill metadata and workflow guidance.
- Local-first LINE Bot starter template.
- LINE webhook server template with static signature gate verification.
- LM Studio local LLM integration defaults.
- SQLite memory implementation and memory policy docs.
- Explicit web-search command flow and safety docs.
- Public hygiene and template verifier scripts.
- Developer quickstart, customization guide, live smoke guide, demo walkthrough, and release checklist.
- GitHub issue templates, PR template, and non-live CI.

## Not Included

- LINE Official Account registration.
- LINE Developers Console login or automation.
- Deployment, hosting, domains, SSL, or tunnel provisioning.
- Real LINE Channel Secret.
- Real Channel Access Token.
- reply token.
- search API key.
- webhook evidence.
- user conversation data.

## Known Limitations

- Signature gate is `STATIC_VERIFIED`, not `RUNTIME_VERIFIED`.
- No live LINE smoke evidence is included.
- Production readiness remains `BLOCKED` until operator evidence exists.
- Not stable.
- Not production ready.
- Live LINE setup is manual and best effort.

## Security Notes

- Do not commit `.env`, tokens, reply tokens, search keys, runtime logs, databases, backups, or private evidence.
- Webhook signature bypass, search SSRF, prompt injection, remote LLM leaks, and memory leaks should be reported privately.
- Remote LLM endpoints are unsafe by default and require manual approval.

## Privacy Notes

- Runtime LINE messages enter the operator-run webhook server.
- SQLite memory can store LINE identifiers, text, summaries, hashes, and event metadata.
- Web search queries may go to third-party search providers when enabled.
- Runtime DBs, logs, backups, and evidence must stay local.

## Expected Readiness BLOCKED Explanation

Fresh template readiness can return `BLOCKED` because real local runtime, public webhook, LINE smoke tests, backup/restore drill, monitoring, and final approval evidence are not included in the public repo.

That `BLOCKED` state is expected for alpha and is not a release blocker when hygiene and template checks pass.
